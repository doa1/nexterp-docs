How Onboarding Works

When a new organization is created, the system runs a 15-step automated setup based on the selected industry. The industry determines which Chart of Accounts template is applied, which modules are enabled, and what POS terminal defaults are configured.

Onboarding flow:

1.  Create Organization (with industry + base currency)
2.  Create HQ Entity (headquarters)
3.  Create GL Books (Local GAAP, IFRS, Tax)
4.  Apply Chart of Accounts (industry-specific template, 200+ accounts)
5.  Create Accounting Periods (2 fiscal years  12 months + closing)
6.  Create Units of Measure (16 default UOMs)
7.  Create Inventory Categories (5 defaults)
8.  Create Cost Centers (4 defaults)
9.  Create Departments (6 defaults)
10. Create Document Sequences (10 sequence patterns)
11. Create Exchange Rates (10+ currency pairs)
12. Create Default POS Terminal + Cash Register (industry settings)
13. Create Walk-in Customer (default client for POS/cash sales)
14. Create Admin User + Roles + Permissions
15. Enable Modules + Create Public Site Config

Steps 115 are fully automated. After setup completes, the client configures industry-specific master data through the UI.

Payroll Run Workflow

The payroll processing cycle follows this workflow:

1. Create Payroll Run      Select pay group, period dates, payment date
2. Process Payroll         System calculates: gross, earnings, deductions, taxes, net
                            - Applies salary components per employee
                            - Calculates statutory deductions (PAYE, NSSF, NHIF, Housing Levy)
                            - Deducts salary advance repayments (if any)
                            - Generates earnings/deductions/taxes breakdown per employee
3. Review & Adjust         HR reviews calculated payslips, makes manual adjustments
4. Approve Payroll         Finance/Director approves the run
5. Post to GL              Journal entries auto-created per component GL mapping
6. Pay Employees           Mark as paid, record payment references

Status flow: draft processed approved paid (or cancelled)
